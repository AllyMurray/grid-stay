# Grid Stay Ideas

Last reviewed: 2026-05-06

This file is a lightweight product backlog for ideas that are not committed roadmap items yet. Keep entries short, practical, and focused on the user problem. When an idea moves into implementation, link the PR or remove it after the feature is shipped and recorded in What's New.

## High-Value Ideas

### Event briefing pack

Create a single briefing page for each booked day that pulls together paddock arrival details, attendee status, garages, hotels, dinner plans, weather, circuit notes, documents, and latest updates.

- User value: reduces pre-event coordination across chat, email, and individual booking pages.
- Possible first version: show the current day detail information as a concise "briefing" section with missing-items prompts.
- Research reference: Sportity's event app focuses on centralised event documents, updates, and push-style communication: https://www.sportity.com/app

### Trip readiness score

Show a clear readiness state for each booking based on key setup items such as status, accommodation choice, arrival time, garage share, dinner plan, costs, and required documents.

- User value: makes it obvious what still needs attention before travelling.
- Possible first version: a simple checklist on My Bookings with "Ready", "Needs attention", and "Optional" sections.

### Email import for bookings and hotels

Allow users to forward booking confirmation or hotel emails so Grid Stay can extract date, circuit, provider, hotel, arrival details, and references.

- User value: reduces manual data entry and improves data quality.
- Possible first version: inbound email address with a review screen before saving imported details.
- Research reference: TripIt builds itineraries from forwarded confirmation emails: https://www.tripit.com/web/free/how-it-works

### Convoy planner

Let users declare travel origin area, departure time, route notes, fuel stops, trailer constraints, and whether they are open to travelling together.

- User value: supports the road-drive and group travel side of the app, not just track-day attendance.
- Possible first version: optional travel card on each event with departure time and notes visible to the group.
- Research reference: Roadtrippers focuses on route planning, stops, and trip coordination: https://roadtrippers.com/about/features/

### Interest polls before booking

Allow users to vote "interested", "maybe", or "not for me" on proposed days before anyone commits.

- User value: helps the group choose days with enough interest before booking.
- Possible first version: add a poll state to user-added events before they become normal available days.
- Research reference: Spond supports invitations, availability, and team planning workflows: https://www.spond.com/en-us/features-for-teams-overview/

## Supporting Ideas

### Targeted event updates

Allow important updates to be posted against a day and shown only to people booked or maybe on that day.

- User value: avoids noisy global notifications while keeping relevant attendees informed.
- Possible first version: day-level updates with an unread badge in the menu or dashboard.

### Better cost ledger

Expand cost splitting into a proper shared ledger with receipts, settlement status, and simplified balances.

- User value: makes garages, hotels, dinner deposits, and shared purchases easier to manage.
- Possible first version: add paid-by, split-between, and settled state to existing costs.
- Research reference: Splitwise centres group expenses around balances and settlement: https://www.splitwise.com/

### Car and kit checklist

Let users keep reusable checklists for car prep, tools, tickets, license, helmet, fuel, and track-specific kit.

- User value: reduces forgotten items and repeated prep work.
- Possible first version: personal checklist templates with per-booking completion state.

### Post-event logbook

Add a post-event note for weather, tyres, setup, issues, hotel feedback, circuit feedback, and photos.

- User value: builds useful history for future bookings and group recommendations.
- Possible first version: private notes plus optional group-visible hotel and circuit tips.

### Hotel suitability tags

Improve hotel records with practical motorsport-focused tags such as trailer parking, secure parking, late check-in, breakfast timing, food nearby, and distance to circuit.

- User value: makes hotel choice easier than a generic rating alone.
- Possible first version: tag filters on the Hotels page and structured feedback prompts.
